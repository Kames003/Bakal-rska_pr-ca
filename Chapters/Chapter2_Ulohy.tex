\chapter{Zoznam úloh zadaných študentovi s~vyjadrením časovej náročnosti}
\label{sec:Ulohy}

% Rozsah: 1-2 strany

Táto kapitola poskytuje prehľad úloh, ktoré mi boli zadané počas odbornej praxe vo firme Railsformers s.r.o. Úlohy boli evidované v~systéme Redmine, kde som priebežne zaznamenával odpracovaný čas a~stav spracovania.

Časové údaje v~tabuľke predstavujú celkovú náročnosť jednotlivých úloh evidovanú v~systéme Redmine. Práca na projekte prebiehala počas dvoch semestrov a~zahŕňala aj činnosti mimo štandardnú pracovnú dobu, ktoré nie sú v~tabuľke zahrnuté -- štúdium dokumentácie nových technológií, iteratívne vylepšovanie používateľského rozhrania, ladenie výkonu databázových dotazov, riešenie okrajových prípadov v~parseri, rozsiahle testovanie v~reálnych podmienkach a~priebežnú optimalizáciu celého systému.

Časové rozloženie práce kopírovalo prirodzený vývoj projektu -- od hardvérovej konfigurácie cez backend až po frontend a~geofencing funkcionalitu. Najväčší podiel času zabrala implementácia frontendu a~backend API, čo je typické pre aplikácie s~komplexným používateľským rozhraním a~rozsiahlou biznis logikou. Nezanedbateľnú časť predstavovali aj optimalizácie a~testovanie, ktoré prebiehali kontinuálne počas celého vývoja a~výrazne prispeli k~stabilite a~kvalite výsledného riešenia.


\section{Stručný popis úloh}
\label{sec:PopisUkolu}

\subsection{Konfigurácia hardvéru}
\label{sec:UlohaHW}

Prvou úlohou bola konfigurácia LoRaWAN infraštruktúry -- gateway RAK7289 V2 a~GPS trackerov SenseCap T1000-B. Práca zahŕňala aktualizáciu firmware gateway z~verzie 2.2.2 na 2.2.14, nastavenie frekvenčného plánu EU868 a~konfiguráciu MQTT integrácie pre odosielanie dát na firemný broker. Trackery boli registrované pomocou OTAA aktivácie s~unikátnymi kľúčmi DevEUI, AppEUI a~AppKey.

\subsection{MQTT broker a~parser}
\label{sec:UlohaMQTT}

Druhá úloha sa zameriavala na pripojenie backendu k~MQTT brokeru \texttt{mqtt.hedurio.com} s~TLS šifrovaním a~implementáciu parsera pre dekódovanie správ z~trackerov. Parser spracováva formát SenseCap, ktorý obsahuje GPS súradnice, stav batérie, teplotu a~ďalšie telemetrické údaje vo formáte Base64/HEX. Súčasťou boli aj validácie pre edge cases ako GPS cold start či drift súradníc.

\subsection{Návrh databázy}
\label{sec:UlohaDB}

Tretia úloha zahŕňala návrh a~implementáciu databázovej schémy v~PostgreSQL s~využitím rozšírení TimescaleDB pre efektívne ukladanie časových radov pozícií a~PostGIS pre geospatial operácie. Schéma obsahuje tabuľky pre zariadenia, pozície, geofence zóny a~body záujmu (POI). Implementoval som indexy, triggery a~optimalizované dotazy pre historické dáta.

\subsection{Backend API}
\label{sec:UlohaAPI}

Štvrtá úloha predstavovala vývoj REST API vo frameworku Flask s~viac ako 28 endpointmi pre správu zariadení, pozícií, geofences a~POI. Súčasťou bola implementácia real-time komunikácie pomocou Socket.IO, ktorá umožňuje okamžité doručovanie nových pozícií do webového klienta bez potreby pollingu.

\subsection{Frontend aplikácia}
\label{sec:UlohaFrontend}

Piata a~časovo najnáročnejšia úloha bola implementácia webovej aplikácie v~React s~využitím Vite ako build nástroja a~Leaflet.js pre mapovú vizualizáciu. Aplikácia podporuje tri režimy zobrazenia -- Live Tracking pre sledovanie v~reálnom čase, Historical Playback pre prehliadanie histórie pohybu a~Geofencing pre správu virtuálnych zón.

\subsection{Geofencing}
\label{sec:UlohaGeofencing}

Šiesta úloha rozšírila aplikáciu o~funkcionalitu geofencing -- možnosť definovať polygonálne virtuálne zóny na mape a~automaticky detekovať vstup, výstup alebo zotrvania zariadenia v~zóne. Implementoval som alert systém s~vizuálnymi notifikáciami a~históriou udalostí.

\subsection{Optimalizácie a~bezpečnosť}
\label{sec:UlohaOptimalizacie}

Siedma úloha sa zamerala na zabezpečenie aplikácie (TLS komunikácia, CORS politika, parameterized SQL queries) a~optimalizáciu výkonu (connection pooling, efektívne indexy, lazy loading). Riešil som tiež edge cases ako race conditions pri súbežnom zápise pozícií a~ošetrenie neplatných GPS dát.

\subsection{Testovanie}
\label{sec:UlohaTesting}

Posledná úloha zahŕňala vývoj GPS simulátora pre generovanie testovacích dát bez potreby fyzických trackerov a~testovanie systému v~reálnych podmienkach. Simulátor umožňuje definovať trasy, rýchlosť pohybu a~simulovať rôzne scenáre vrátane výpadkov signálu.
