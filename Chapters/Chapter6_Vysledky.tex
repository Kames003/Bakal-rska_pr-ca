\chapter{Dosažené výsledky v průběhu odborné praxe a její celkové zhodnocení}
\label{sec:Vysledky}

% Rozsah: 3-4 strany

V~tejto kapitole zhodnocujem výsledky dosiahnuté počas odbornej praxe, prezentovaním metriky projektu, spätnú väzbu od mentora a~reflexiu nad tým, čo by som v~budúcnosti urobil inak.

\section{Súhrn implementovanej funkcionality}
\label{sec:Funkcionalita}

Počas šiestich mesiacov odbornej praxe som implementoval kompletný GPS tracking systém, ktorý pokrýva celý životný cyklus dát -- od prijatia surových GPS súradníc z~LoRaWAN trackerov až po ich vizualizáciu v~interaktívnej webovej aplikácii.

\subsection{Real-time tracking}
\label{sec:VysledkyRealtime}

Jadrom systému je real-time sledovanie pozícií pomocou WebSocket komunikácie. Každá nová GPS pozícia prijatá cez MQTT broker je okamžite distribuovaná všetkým pripojeným klientom. Používateľ vidí pohyb trackerov na mape v~reálnom čase bez nutnosti manuálneho obnovovania stránky.

\subsection{Historical playback}
\label{sec:VysledkyHistorical}

Režim Historical umožňuje prehliadanie histórie pohybu za zvolené časové obdobie. Implementoval som timeline slider, ktorý umožňuje plynulú animáciu trasy, a~heatmap vizualizáciu zobrazujúcu intenzitu pobytu na jednotlivých miestach. Táto funkcionalita je užitočná pre analýzu vzorcov pohybu a~identifikáciu frekventovaných oblastí.

\subsection{Geofencing a~POI}
\label{sec:VysledkyGeofencing}

Geofencing modul umožňuje vytváranie virtuálnych zón (polygónov) na mape s~automatickým generovaním alertov pri vstupe, výstupe alebo dlhšom pobyte zariadenia v~zóne. Komplementárne k~tomu som implementoval Points of Interest (POI) s~funkciou nearby search, ktorá využíva PostGIS pre efektívne priestorové dotazy.

\subsection{Device management}
\label{sec:VysledkyDevices}

Správa zariadení zahŕňa prehľad všetkých registrovaných trackerov so štatistikami, monitorovanie stavu batérie s~odhadom zostávajúcej výdrže, a~systém tagov pre kategorizáciu zariadení (napr. security, staff, VIP).

\section{Technická kvalita}
\label{sec:TechnickaKvalita}

Pri implementácii som dbal na dodržiavanie osvedčených praktík softvérového inžinierstva:

\begin{itemize}
    \item \textbf{Modulárna architektúra} -- Oddelenie MQTT klienta, parsera, databázovej vrstvy a~API umožňuje nezávislý vývoj a~testovanie jednotlivých komponentov.
    \item \textbf{Separation of concerns} -- Frontend komponenty sú rozdelené podľa zodpovednosti (MapView, Sidebar, Timeline, Geofencing).
    \item \textbf{Škálovateľný databázový návrh} -- Použitie TimescaleDB hypertables umožňuje efektívne spracovanie veľkého objemu časových dát.
    \item \textbf{Automatizované triggery} -- Databázové triggery automaticky generujú geometrické hodnoty a~kontrolujú geofence eventy.
    \item \textbf{Bezpečnosť} -- TLS šifrovanie MQTT komunikácie, parameterized SQL queries pre prevenciu SQL injection, CORS konfigurácia.
\end{itemize}

\section{Čo by som urobil inak}
\label{sec:CoJinak}

S~odstupom času a~nadobudnutými skúsenosťami identifikujem niekoľko oblastí, kde by som v~budúcnosti postupoval odlišne.

\subsection{Skoršie real-world testovanie}
\label{sec:RealWorldTesting}

Väčšinu vývoja som realizoval so simulovanými GPS dátami, čo umožnilo rýchly iteratívny vývoj. Avšak niektoré edge cases, ako GPS drift pri slabom signáli alebo battery stuck bug (keď tracker reportuje konštantnú hodnotu batérie), som objavil až pri testovaní s~reálnymi zariadeniami. Skoršia integrácia reálnych trackerov by tieto problémy odhalila skôr.

\subsection{Automatizované testy}
\label{sec:AutomatizovaneTesty}

Projekt aktuálne nemá formálnu test suite. Implementácia unit testov pre parser a~integration testov pre API by významne uľahčila refactoring a~pridávanie nových funkcionalít. Pri komplexnejších zmenách som musel manuálne overovať, či som nerozbil existujúcu funkcionalitu.

\subsection{Dokumentácia počas vývoja}
\label{sec:DokumentaceBehem}

Retrospektívne písanie dokumentácie pre bakalársku prácu bolo náročnejšie, než by bolo priebežné dokumentovanie rozhodnutí. Odporúčam budúcim študentom viesť si denník vývoja s~poznámkami o~dôležitých rozhodnutiach a~ich zdôvodneniach.

\subsection{CI/CD pipeline}
\label{sec:CICD}

Implementácia automatizovaného CI/CD pipeline (napr. GitHub Actions) by zvýšila kvalitu kódu vďaka automatickým kontrolám pri každom commite. Aktuálne deployment prebieha manuálne cez Docker Compose.

\section{Celkové zhodnotenie}
\label{sec:CelkoveZhodnotenie}

Napriek identifikovaným oblastiam na zlepšenie považujem projekt za úspešný. Podarilo sa mi implementovať funkčný GPS tracking systém, ktorý spĺňa všetky požiadavky zadania a~prináša praktickú hodnotu pre potenciálne využitie na eventoch. Získané skúsenosti s~IoT technológiami, time-series databázami a~real-time webovými aplikáciami sú cenným prínosom pre moju ďalšiu kariéru v~IT.
